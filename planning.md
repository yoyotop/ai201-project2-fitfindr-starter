### Tool 1: search_listings

What it does:
Searches the mock thrift listings dataset for items matching the user's description, size, and maximum price. Results are filtered and ranked by keyword relevance.

Input parameters:

description (str): Keywords describing the item the user wants.
size (str): Optional size filter such as S, M, or L.
max_price (float): Optional maximum price filter.

What it returns:
A list of listing dictionaries sorted by relevance. Each listing contains fields such as title, description, category, style_tags, size, condition, price, brand, colors, and platform.

What happens if it fails or returns nothing:
The tool returns an empty list. The agent informs the user that no listings matched the search criteria and stops the workflow.
---

### Tool 2: suggest_outfit

What it does:
Uses an LLM to generate outfit suggestions based on a thrifted item and the user's wardrobe.

Input parameters:

new_item (dict): The selected thrift listing.
wardrobe (dict): The user's wardrobe containing available clothing items.

What it returns:
A string containing one or more outfit recommendations.

What happens if it fails or returns nothing:
If the wardrobe is empty, the tool provides general styling advice for the item. If an error occurs, it returns a descriptive error message instead of crashing.
---

### Tool 3: create_fit_card

What it does:
Uses an LLM to generate a social-media-style caption based on the outfit suggestion and thrifted item.

Input parameters:

outfit (str): Outfit recommendation generated by suggest_outfit().
new_item (dict): The selected thrift listing.

What it returns:
A short caption suitable for Instagram or TikTok.

What happens if it fails or returns nothing:
If outfit information is missing, the tool returns an explanatory error message instead of raising an exception.

## Planning Loop

**How does your agent decide which tool to call next?**
The agent begins by creating a session dictionary for the interaction. It parses the user's query using regular expressions to extract a description, size, and maximum price.

The agent calls search_listings() using the parsed information. If no listings are found, an error message is stored in the session and the workflow ends immediately.

If results are found, the first result is selected and stored as selected_item. The agent then calls suggest_outfit() using the selected item and the user's wardrobe. The returned outfit suggestion is stored in the session.

Next, the agent calls create_fit_card() using the outfit suggestion and selected item. The generated caption is stored in the session.

When all steps are complete, the session dictionary is returned to the interface and displayed to the user.
---

## State Management

**How does information from one tool get passed to the next?**
The application stores all information inside a session dictionary. The session tracks the original query, parsed parameters, search results, selected item, wardrobe, outfit suggestion, fit card, and any error messages.

Data flows between tools through the session dictionary. The selected item returned from search_listings() becomes the input to suggest_outfit(). The outfit suggestion returned from suggest_outfit() becomes the input to create_fit_card(). This allows information to move between tools without requiring the user to enter it again.

## Error Handling

For each tool, describe the specific failure mode you're handling and what the agent does in response.

search_listings
Failure mode: No results match the query.
Agent response: Returns an empty list. The agent stores an error message in the session, informs the user that no matching listings were found, and stops the workflow.

suggest_outfit
Failure mode: Wardrobe is empty.
Agent response: Generates general styling advice for the selected item instead of failing.

create_fit_card
Failure mode: Outfit input is missing or incomplete.
Agent response: Returns a descriptive error message explaining that outfit information is missing instead of crashing.

## A Complete Interaction (Step by Step)

Example user query: "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

Step 1:
The agent parses the query and extracts:

Description: "vintage graphic tee"
Max price: 30
Size: None

The agent calls search_listings().

Step 2:
search_listings() returns matching thrift listings. The top result is selected and stored in session["selected_item"].

Step 3:
The agent calls suggest_outfit() using the selected item and the user's wardrobe. The returned outfit recommendation is stored in session["outfit_suggestion"].

Step 4:
The agent calls create_fit_card() using the outfit recommendation and selected item. The generated caption is stored in session["fit_card"].

Final output to user:
The user sees the selected thrift listing, a complete outfit recommendation, and a social-media-ready fit card caption.
